# Task 3 — Проектирование целевой архитектуры и оценка рисков

## Описание

Целевая архитектура системы «Будущее 2.0» на горизонте трёх лет, спроектированная по C4-модели, и комплексная оценка рисков трансформации.

## Артефакты

| Файл | Описание |
|------|----------|
| `c4-context.drawio` | C4 Context Diagram — общий ландшафт системы с внешними актёрами, доменами и legacy-системами |
| `c4-container.drawio` | C4 Container Diagram — целевая архитектура с детализацией контейнеров по доменам |
| `risk-map.md` | Карта рисков: 15 рисков (архитектурные, организационные, технологические) с оценкой вероятности и влияния |
| `risk-management-plan.md` | План управления рисками с приоритизацией, ответственными, метриками и процессом мониторинга |

## Ключевые решения целевой архитектуры

1. **Event-Driven Architecture** — Apache Kafka как центральная событийная платформа вместо Camel ESB
2. **Data Mesh** — доменные команды владеют своими данными, портал самообслуживания для аналитиков
3. **Managed Kubernetes** — единая платформа оркестрации в Yandex Cloud
4. **Поэтапная миграция** — anti-corruption layers для сосуществования legacy и новых систем
