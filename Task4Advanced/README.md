# Task 4 — Моделирование домена и интеграций

## Описание

Доменная модель системы «Будущее 2.0» по методологии Domain-Driven Design (DDD) с событийной архитектурой интеграций.

## Артефакты

| Файл | Описание |
|------|----------|
| `bounded-contexts.drawio` | Context Map — 9 bounded contexts с DDD-отношениями (Customer/Supplier, ACL, Shared Kernel, OHS, Conformist) |
| `event-storming.drawio` | Event Storming диаграмма — команды, агрегаты, события, политики и hot spots по 6 доменам |
| `aggregates.md` | 14 агрегатов с границами, инвариантами, идентификаторами и жизненными циклами |
| `events.md` | Каталог из 33 доменных событий с контрактами (JSON-схемы) |
| `justification.md` | Обоснование событийного подхода с количественным сравнением текущей и целевой архитектуры |

## Bounded Contexts

1. **Управление пациентами** — регистрация, профили, мед. карты, согласия
2. **Клинические операции** — расписание, приёмы, назначения
3. **Медицинские исследования и ИИ** — лабораторные исследования, ML-inference
4. **Финансовые операции** — счета, платежи, возвраты
5. **Кредитование** — заявки, скоринг, договоры, графики платежей
6. **Аналитика и BI** — датапродукты, отчёты, портал самообслуживания
7. **Управление персоналом** — сотрудники, квалификации, сертификаты
8. **Инвентаризация** — оборудование, медикаменты, расходные материалы
9. **Платформа данных** — каталог данных, метаданные, SLA
